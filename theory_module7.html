Экспертные системы поддержки принятия решений

ОПРЕДЕЛЕНИЯ

  • Интеллект – способность мышления и рационального познания, которые реализуются посредством приобретения и применения знаний для решения задач.
  • Искусственный интеллект – свойство автоматизированных систем брать на себя отдельные функции интеллекта человека.
  • Система искусственного интеллекта (СИИ) – комплекс программных, лингвистических и логико-математических средств для поддержки деятельности человека.
  • Экспертные системы – направление исследований в области искусственного интеллекта по созданию вычислительных систем, умеющих принимать решения, схожие с решениями экспертов.

ЗАДАЧИ ЭКСПЕРТНЫХ СИСТЕМ

  • Создаются для решения практических задач в узкоспециализированных областях
  • Позволили привлечь внимание к исследованиям в области ИИ

КЛАССИФИКАЦИЯ ЭКСПЕРТНЫХ СИСТЕМ

По степени интеграции:

  • Автономные
  • Гибридные (интегрированные)

По связи с реальным временем:

  • Статические
  • Квазидинамические
  • Динамические

НЕДОСТАТКИ ЭС

  • Трудоемкость передачи "глубоких" знаний
  • Неспособность предоставить осмысленные объяснения
  • Описывают только последовательность шагов
  • Неспособность к самообучению

КОМПОНЕНТЫ ЭКСПЕРТНЫХ СИСТЕМ

База знаний (БЗ)

Функция: Хранит формализованные знания экспертов

Состав:

  • Факты (например, "Температура пациента = 38°C")
  • Правила (например, "ЕСЛИ температура > 37°C, ТО возможна инфекция")
  • Онтологии (описание связей между понятиями)

Типы представления знаний:

  • Продукционные модели (правила "ЕСЛИ-ТО")
  • Семантические сети (графы связей)
  • Фреймы (шаблоны для описания объектов)

Механизм логического вывода

Функция: Обрабатывает данные для принятия решений

Принципы работы:

  • Прямая цепочка (от фактов к выводам)
  • Обратная цепочка (от гипотезы к фактам)

Пользовательский интерфейс

Функция: Обеспечивает взаимодействие с пользователем

Формы:

  • Естественно-языковой интерфейс (чат-боты)
  • Графический интерфейс (формы, диаграммы)
  • API (интеграция с другими системами)

Подсистема объяснений

Функция: Показывает, как система пришла к выводу

Подсистема приобретения знаний

Функция: Позволяет обновлять базу знаний

Методы:

  • Работа с экспертом (интервью, анкетирование)
  • Машинное обучение (автоматическое извлечение знаний)

Дополнительные компоненты (в продвинутых ЭС)

  • База данных (хранение статистики)
  • Модуль самообучения
  • Подсистема верификации

Примеры архитектур ЭС

Тип Описание Пример
Моноагентные Одна база знаний + один движок вывода Медицинская диагностика
Мультиагентные Несколько взаимодействующих ЭС Управление производственными процессами
Гибридные Комбинация ЭС с нейросетями/ML Финансовый анализ + прогнозирование

Применение экспертных систем

  • Медицина (диагностика, например, MYCIN)
  • Финансы (кредитный скоринг)
  • Техподдержка (определение неисправностей)
  • Юриспруденция (анализ прецедентов)

ЗАКЛЮЧЕНИЕ

Структура ЭС обеспечивает имитацию человеческого экспертного мышления за счёт комбинации базы знаний, механизма вывода и интерфейсов. Современные тенденции включают интеграцию с искусственным интеллектом и большими данными.
